Справка по блоку

Блок FMIimport реализует импорт модели по стандарту FMI 1.0 Model Exchange.
Описание и реализация функций модели передается с помощью архива с расширением .fmu,
в котором содержатся .dll - файл с реализацией функций модели и .xml файл с описанием переменных модели.
Архив распаковывается в папку с проектом. В данном блоке реализована возможность вывода любой переменной
в качестве выходного сигнала, даже если ее тэг не "input". 

Для работы данного блока необходимо указать адрес .fmu архива, указать количество входов
(из .xml файла) количество выходов и названия переменных (из .xml файла) которым будут соответствовать выходы.

Переменные строкового типа не поддерживаются (ввиду того, что входные и выходные сигналы - вещественные числа). 

Входы:

Input_0 .. N - входные скалярные сигналы, где N-1 - количество входов.

Выходы: 

Output_0 .. N - выходные скалярные сигналы, где N-1 - количество выходов

Свойства:

Адрес FMU - адрес архива .fmu

Количество входов - устанавливает количество входных скалярных сигналов
(необходимо соответствие с количеством входов из .xml файла)

Количество выходов - устанавливает количество выходных скалярных сигналов
(необходимо соответствие с количеством правильно указанных имен переменных из свойств Имя переменной выхода 1 .. 5)

Имя переменной выхода 1 .. 5 - имя переменной, которой будет соответствовать один из выходов
(название берется из файла .xml; если название неверное или пустое, то выход не инициализируется)

Примечание: в блоке возможна поддержка вывода переменных строкового типа как параметры блока.
